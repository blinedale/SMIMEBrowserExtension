How it works

The plugin will ignore messages that are not signed with S/MIME.

If a message has a valid signature that matches the message content and the 'From' address matches the signer's email in the certificate, that email will be marked as verified.

Message that look signed but where the verification fails for some reason will be marked with a warning.

Please note that any email in a thread has to be opened completely to trigger a verification; we do not verify emails that are collapsed or hidden in previous thread history.

Also, we currently do not check if a certificate has been revoked while verifying a message signature. Messages signed with a revoked certificate will currently be shown as verified if everything else checks out. We are working on adding this functionality.
